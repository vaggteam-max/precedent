Detection pipeline

Stage 1 — regex cue prefilter (decision / commitment / re-litigation), zero cost per message. Stage 2 — LLM extraction of title, rationale, rejected alternatives with confidence threshold. Stage 3 — human confirmation card. Nothing is logged without a click.

Re-litigation guard

Question-phrased cues trigger a registry precedent check before classification. On a match: interject with the original decision, decider and permalink.

Grounded answerer

Registry search + RTS context assembled into cited answers (“Decision #5, decided by …, thread ↗”). Declines to invent when nothing matches.

Nudge scheduler

Hourly sweep of open commitments; owners receive a DM before due dates slip. Status lifecycle: open → nudged → done.

Registry data model

Decision: title · rationale · alternatives · decider · status · permalink. Commitment: owner · action · due date · status · permalink.

LLM providers REST

Gemini Flash primary with automatic model fallback on 503/429 → any OpenAI-compatible endpoint → Anthropic. Provider-agnostic JSON contract.

Real-Time Search API

assistant.search.context — workspace-wide context enrichment for answers, with graceful degradation.

Qualifying tech — RTS API

SQLite registry node:sqlite

Embedded, zero native dependencies. Tables: decisions, commitments. Every row carries a permalink to its source thread — answers cite receipts.
